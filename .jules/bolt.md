## 2024-05-24 - Python math.hypot Performance for Root Mean Square
**Learning:** For calculating the RMS of an audio chunk (represented as unpacked integer samples), using `math.hypot(*samples) / math.sqrt(count)` is significantly faster (approx 3.4x to 7.5x depending on chunk size) than `math.sqrt(sum(s * s for s in samples) / count)`. `math.hypot` is implemented in C and avoids the overhead of evaluating a Python generator expression on every audio frame. While argument unpacking (`*samples`) does have overhead and stack limit limits in older versions, for chunks up to a few thousand elements in modern CPython (3.13), this is safe and much more performant for hot loops.
**Action:** Always prefer C-optimized functions like `math.hypot` for heavy number crunching inner loops when `numpy` is not available, avoiding Python-level looping and generator expressions.
