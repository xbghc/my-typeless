## 2024-05-24 - Navigation Tags and Stateful ARIA Labels
**Learning:** `<a>` tags used for client-side routing or interaction without an `href` attribute are completely skipped by keyboard navigation (Tab key). Furthermore, icon-only buttons that toggle state (like a password visibility eye icon) must not only have an initial `aria-label`, but must dynamically update that label (e.g., from "Show password" to "Hide password") to give screen readers accurate context of the current state, otherwise users won't know the result of their action.
**Action:** Always use `<button>` tags for interactive elements that don't perform native browser navigation. When using icon-only buttons that toggle state, ensure JavaScript updates the `aria-label` attribute in sync with the visual state change.
