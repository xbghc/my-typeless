## 2024-05-24 - Dynamic ARIA Labels for Stateful Toggle Buttons
**Learning:** For stateful icon-only toggle buttons (like password visibility toggles), setting an initial `aria-label` in HTML is insufficient for maintaining accessibility. When the visual state of the button changes (e.g., from an eye icon to a slashed-eye icon), screen readers need to be informed of the corresponding action change. If the `aria-label` doesn't update, the screen reader will misleadingly announce "Show password" even when the password is already visible.
**Action:** Always ensure that JavaScript logic handling state toggles also dynamically updates the `aria-label` attribute on the triggering button to match the new visual state and expected next action (e.g., swapping between "Show password" and "Hide password").
